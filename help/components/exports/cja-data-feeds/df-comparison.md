# Comparación de fuentes de datos en Customer Journey Analytics y Adobe Analytics

Las fuentes de datos tanto en Customer Journey Analytics como en Adobe Analytics permiten exportar datos sin procesar a plataformas de terceros. Si anteriormente utilizaba fuentes de datos en Adobe Analytics, utilice la siguiente información para comprender las diferencias en las funciones disponibles, el comportamiento del procesamiento de datos y mucho más.

## Comparar funcionalidad de fuentes de datos

| **Opciones y conceptos de configuración** | **Descripción** | **Customer Journey Analytics** | **Adobe Analytics** |
|---------|----------|---------|---------|
| Entrada de datos | El tipo de datos que se pueden recopilar e incluir en las fuentes de datos. | Admite entrada de datos en canales múltiples, incluidos datos web, datos del centro de llamadas, datos del punto de venta, etc. | Solo admite la entrada de datos web y móviles. |
| Procesamiento de datos | A menudo, los datos deben procesarse antes de que sean útiles para la creación de informes y este procesamiento afecta a la funcionalidad de las fuentes de datos. Los datos se procesan en diferentes etapas, según si utiliza Customer Journey Analytics o Adobe Analytics. | Los datos se procesan en el momento del informe y, por lo tanto, muchas funciones de creación de informes de Customer Journey Analytics se pueden utilizar para cambiar los datos históricos, como la vinculación, los campos derivados, la preparación de datos, la segmentación y la ventana de creación de informes. | Los datos se procesan en el momento de la recopilación y, por lo tanto, las funciones de creación de informes como las reglas de procesamiento, las reglas de VISTA, la vinculación y la segmentación no afectan a los datos históricos. |
| Ventana de fuente de datos/Frecuencia de entrega | Determina:<ul><li>La frecuencia con la que se envía la fuente de datos</li><li>Período de tiempo incluido en la fuente de datos<p>En otras palabras, una vez que finaliza la ventana de fuente de datos y se envía la fuente de datos, se inicia una nueva ventana de fuente de datos.</p></li></ul><p>Las opciones disponibles son las siguientes:</p><ul><li>**Diario**: las fuentes contienen datos de un día completo, de medianoche a medianoche en el huso horario del grupo de informes. Además, las fuentes de 15 minutos son posibles, pero no están disponibles de forma predeterminada. </p><p>Las visitas que llegan tarde se pueden configurar según la **[!UICONTROL ventana retrospectiva]**.</p><p>Ejemplo: 1 hora</p> |
| Inclusión de visitas que llegan tarde | Las visitas que llegan tarde son visitas que llegan durante la ventana de fuente de datos actual, pero tienen una marca de tiempo que cae en una ventana de fuente de datos anterior (por ejemplo, a través de visitas con marca de tiempo o fuentes de datos). Cuando las visitas llegan tarde, se agrupan en el siguiente archivo de fuente de datos que se envía, aunque las marcas de tiempo pertenezcan a una ventana anterior. | Las visitas que llegan tarde y que se producen dentro de la **[!UICONTROL ventana de informes]** siempre se incluyen. <p>Las visitas se reordenan automáticamente según la marca de tiempo en la salida (por ejemplo, las visitas tardías pueden aparecer antes que las visitas a tiempo si sus marcas de tiempo son anteriores). No hay fuentes de cambios, por lo que los valores originales se mantienen. | Las visitas que llegan tarde se pueden incluir o excluir y se pueden configurar con la opción de configuración **[!UICONTROL Visitas que llegan tarde]**.<p>La ventana retrospectiva para estas visitas se configura a través de la opción de configuración **[!UICONTROL Ventana retrospectiva]** disponible para este propósito específico.</p><p>Las visitas se muestran en el orden en que se reciben; no se reordenan según la marca de tiempo.</p> |
| Visitas fuera de orden | H | Las visitas desordenadas se evalúan por persona; CJA puede aceptar patrones de llegada de flujo continuo + lotes y reordena una vez que los datos llegan a la secuencia de tiempo adecuada. CJA no espera a confirmar que tiene todos los datos para la ventana de creación de informes antes de enviarlos; envía lo que llegó durante la hora/día (que puede incluir visitas que llegan tarde y cuyas marcas de hora caen antes, dentro de la ventana de creación de informes). <p>**Importante**: el consumidor final de los datos de la fuente de datos debe poder controlar las marcas de tiempo que estén desordenadas, ya que los datos de la fuente de datos pueden incluir visitas tardías con marcas de tiempo anteriores a la ventana de la fuente de datos.</p> | |
| Ventana retrospectiva (solo Adobe Analytics) | El límite para visitas tardías o fuera de orden que se entregan fuera de la **[!UICONTROL frecuencia de entrega]** seleccionada (como **[!UICONTROL Hora]** o **[!UICONTROL Día]**). Las visitas deben comenzar después de este límite para que se incluyan; no se incluyen las visitas que comienzan antes del límite y finalizan dentro de la ventana retrospectiva.<p>No se utiliza para persistencia, sesiones o dimensiones (estas se incluyen en los datos sin procesar que se recopilan).</p> | Incluido en la **[!UICONTROL ventana de informes]**. | Admitido<p>Ejemplo: 23 horas</p> |
| Ventana Informes (solo Customer Journey Analytics) | La ventana de tiempo que incluye:<ul><li>La ventana de fuente de datos actual (la hora o el día más recientes, como se seleccionó en el campo **[!UICONTROL Frecuencia de entrega]**).</li><li>Cantidad de tiempo especificada antes de la ventana de fuente de datos actual que permite visitas tardías o desordenadas.</li></ul> <p>Necesario para sesiones, persistencia y segmentos.</p><p>No se utiliza para dimensiones. Las dimensiones se controlan por dimensión en función de su asignación y caducidad. Las retrospectivas de Dimension no pueden superar la ventana del sistema de informes.</p> | Admitido<p>Ejemplo: 24 horas</p> | N/D<p>Consulte &quot;Ventana retroactiva&quot;</p> |

Sin embargo, dado que la ventana de creación de informes también incluye las visitas individuales 4 y 3 (que son visitas que llegan tarde con marcas de tiempo de una ventana de fuente de datos anterior), también se incluyen en la ventana de fuente de datos actual.<p>Las visitas se reordenan en la fuente de datos según su marca de tiempo de la siguiente manera: visita individual 3, visita individual 4, visita individual 5.</p> | Solo la visita individual 5 se encuentra en la ventana de fuente de datos. Sin embargo, como se configura una retrospectiva que incluye las visitas individuales 4 y 3 (que son visitas que llegan tarde con marcas de tiempo de una ventana de fuente de datos anterior), también se incluyen en la ventana de fuente de datos actual. (Si no se configurara una retrospectiva, solo se incluiría la visita 5 en la fuente de datos). <p>Las visitas se muestran en la fuente de datos en el orden en que se recibieron, de la siguiente manera: Visita 4, Visita 3, Visita 5.</p> |
| Visitas que llegan tarde/Visitas desordenadas | (En AA: Datos desordenados significa: Desordenados por visitante. En CJA: significa desordenado por persona. Si nos envía datos que no están ordenados por persona, lo estaría si configura la marca de tiempo. Puede establecer la marca de tiempo de dos maneras: puede hacer que Adobe establezca la marca de tiempo en función del momento en que recibimos los datos. O puedes configurarlo tú mismo. Si está configurando las marcas de tiempo y nos envía datos desordenados, esto complica las cosas en AA. En AA, los datos deben llegar en orden por visitante. Necesitamos el orden correcto de los eventos. Pero en CJA, no importa qué marcas de tiempo haya en los datos. CJA no asigna una marca de tiempo a una visita. Esto se hace en sentido ascendente. CJA reordena los datos una vez que llegan, de modo que todo esté en la secuencia de tiempo adecuada. A continuación, podemos realizar el procesamiento del tiempo del informe. Esto significa que puede tener datos de flujo continuo y datos por lotes. No importa. No importa. En el momento en que llega, lo reordenamos y se pone en orden por persona como resultado. Por lo tanto, en CJA le proporcionaremos todos los datos recibidos el último día o la última hora, pero se limitan al principio de la ventana de creación de informes. Lo más probable es que una gran parte de los datos que se obtienen en un día o una hora pertenezca a ese día o esa hora. Si todo lo que hicieras fueran datos por lotes de un centro de llamadas, entonces eso es lo que conseguirías. En CJA, los datos pueden entrar y no importa cuándo entren. Por lo tanto, el cliente de la fuente de datos debe ser capaz de manejar esto de su lado. Así que dondequiera que coloquen los datos, tiene que manejar el hecho de que las marcas de tiempo podrían estar en todas partes. Esto puede ser un desafío para algunos clientes. Ellos necesitan saber esto. Debe poder gestionar datos desordenados por persona. No importa entre la gente. ) Visitas que deberían haberse incluido en una fuente de datos anterior, pero que por alguna razón llegaron tarde (por ejemplo, mediante visitas con marca de hora o fuentes de datos). <p>Estas visitas que llegan tarde se incluyen en la fuente de datos actual en el momento en que llegan, aunque las marcas de tiempo estén dentro de un intervalo de fuente de datos anterior. Cada vez que una fuente de datos procesa la información, observa las visitas que llegan tarde y las procesa en el siguiente archivo de fuente de datos que se envía.</p> | Las visitas que llegan tarde y que se producen dentro de la **[!UICONTROL ventana de informes]** siempre se incluyen. <p>La ventana retrospectiva para estas visitas que llegan tarde se controla mediante la opción de configuración **[!UICONTROL Ventana de informes]**.</p><p>Las visitas se reordenan automáticamente en función de las marcas de tiempo; los valores originales se conservan (sin fuente de cambios).</p> | Se puede incluir o excluir.
